# Platform Signature Verification System

## Overview

The Platform Signature Verification System is a critical security component of SecureStego that ensures only media files generated by our platform can be extracted. This system uses cryptographic HMAC-SHA256 signatures to verify the authenticity of steganographic media before allowing message extraction.

### Purpose

This system is **NOT** a visible watermark or copyright protection mechanism. It is an internal platform verification system that:

- **Prevents unauthorized extraction** of media created by external tools
- **Ensures platform integrity** by verifying all media originated from SecureStego
- **Detects tampering** through cryptographic signature verification
- **Provides audit trail** through comprehensive logging

### Key Security Features

- **HMAC-SHA256 Cryptographic Verification**: Uses industry-standard HMAC with SHA256 for signature generation and validation
- **Platform Secret Key**: Stored in environment variables, never exposed to frontend
- **Payload Binding**: Signatures are cryptographically bound to the specific payload data
- **Media Type Validation**: Prevents cross-media signature replay attacks
- **Version Control**: Enables backward compatibility and future upgrades

## Signature Generation Workflow

### Step 1: Create Signature Structure

The signature structure contains platform metadata:

```json
{
  "platform": "SecureStego",
  "version": "1.0",
  "signature": "base64_encoded_hmac_sha256",
  "createdAt": "2024-01-01T00:00:00Z",
  "mediaType": "image"
}
```

### Step 2: Generate HMAC-SHA256

The signature is generated using:

```python
data_to_sign = {
    "platform": "SecureStego",
    "version": "1.0.0",
    "mediaType": "image",
    "createdAt": "ISO-8601 timestamp",
    "payloadHash": "SHA256 hash of payload data"  # Binds signature to payload
}

signature = HMAC_SHA256(
    key=PLATFORM_SECRET_KEY,
    message=json.dumps(data_to_sign, sort_keys=True)
)
```

### Step 3: Serialize and Inject

The signature is serialized to binary and combined with the encrypted payload:

```
[signature_length (4 bytes)][signature_binary][encrypted_payload_binary]
```

This combined payload is then embedded using the appropriate steganography method.

## Signature Validation Workflow

### Step 1: Extract Combined Payload

The steganography extraction process retrieves the combined payload from the media.

### Step 2: Separate Signature

The signature is separated from the encrypted payload:

```python
signature_length = int.from_bytes(combined_payload[:4], byteorder='big')
signature_binary = combined_payload[4:4 + signature_length]
encrypted_payload = combined_payload[4 + signature_length:]
```

### Step 3: Deserialize Signature

The signature binary is deserialized to a dictionary:

```python
signature_dict = json.loads(signature_binary.decode('utf-8'))
```

### Step 4: Verify HMAC-SHA256

The signature is verified by recomputing the HMAC:

```python
# Reconstruct data that was signed
data_to_verify = {
    "platform": signature_dict["platform"],
    "version": "1.0.0",
    "mediaType": signature_dict["mediaType"],
    "createdAt": signature_dict["createdAt"],
    "payloadHash": SHA256(encrypted_payload).hexdigest()
}

# Compute HMAC
computed_signature = HMAC_SHA256(
    key=PLATFORM_SECRET_KEY,
    message=json.dumps(data_to_verify, sort_keys=True)
)

# Compare using constant-time comparison
is_valid = hmac.compare_digest(computed_signature, provided_signature)
```

### Step 5: Validate Platform Identity

```python
if signature_dict["platform"] != "SecureStego":
    raise PlatformMismatchError("Not generated by SecureStego")
```

### Step 6: Validate Version Compatibility

```python
if signature_dict["version"] not in SUPPORTED_VERSIONS:
    raise VersionMismatchError("Unsupported signature version")
```

### Step 7: Validate Media Type

```python
if signature_dict["mediaType"] != actual_media_type:
    raise MediaTypeMismatchError("Media type mismatch")
```

### Step 8: Return Result

If all validations pass, the encrypted payload is returned for decryption. If any validation fails, an error is returned and extraction stops.

## HMAC Verification Process

### Why HMAC-SHA256?

HMAC (Hash-based Message Authentication Code) with SHA256 provides:

1. **Cryptographic Integrity**: Ensures the signature has not been tampered with
2. **Authentication**: Proves the signature was generated with the correct secret key
3. **Collision Resistance**: SHA256 is resistant to collision attacks
4. **Performance**: Fast computation suitable for real-time verification
5. **Industry Standard**: Widely adopted and well-studied

### Security Properties

- **Key Secrecy**: The platform secret key is never exposed in signatures or logs
- **Constant-Time Comparison**: Prevents timing attacks during verification
- **Payload Binding**: Signature includes payload hash to prevent payload substitution
- **Timestamp**: Prevents replay attacks by including creation time
- **Media Type Binding**: Prevents cross-media signature reuse

## API Documentation

### Platform Verification Service

#### prepare_payload_for_embedding()

Prepares payload for embedding by adding platform signature.

**Parameters:**
- `encrypted_payload_binary` (bytes): Binary encrypted payload
- `media_type` (str): Type of media (image, video, audio, document)

**Returns:**
- `Tuple[bytes, Dict]`: Combined payload and signature info

**Example:**
```python
combined_payload, signature_info = platform_verification_service.prepare_payload_for_embedding(
    encrypted_payload_binary=payload_binary,
    media_type="image"
)
```

#### extract_and_verify_signature()

Extracts and verifies platform signature from combined payload.

**Parameters:**
- `combined_payload` (bytes): Combined payload extracted from media
- `actual_media_type` (str): Actual media type of the file

**Returns:**
- `Tuple[bytes, Dict]`: Encrypted payload and verification result

**Raises:**
- `MissingSignatureError`: Signature not found
- `InvalidSignatureError`: Invalid signature format
- `TamperedSignatureError`: HMAC verification failed
- `VersionMismatchError`: Unsupported version
- `PlatformMismatchError`: Platform identity mismatch
- `MediaTypeMismatchError`: Media type mismatch

**Example:**
```python
try:
    payload_binary, verification_result = platform_verification_service.extract_and_verify_signature(
        combined_payload=combined_payload,
        actual_media_type="image"
    )
except SignatureVerificationError as e:
    # Handle verification failure
    error_response = platform_verification_service.get_error_response(e)
```

#### get_error_response()

Generates standardized error response for verification failures.

**Parameters:**
- `error` (SignatureVerificationError): The verification error

**Returns:**
- `Dict`: Standardized error response

**Example:**
```python
error_response = {
    "success": False,
    "error": "This media was not generated by SecureStego.",
    "errorType": "TamperedSignatureError"
}
```

## Security Considerations

### 1. Secret Key Management

- **Storage**: Platform secret key stored in environment variables (`.env` file)
- **Generation**: Use cryptographically secure random key generation:
  ```bash
  python -c "import secrets; print(secrets.token_hex(32))"
  ```
- **Rotation**: Regular key rotation recommended for production
- **Backup**: Secure backup of keys required for recovery
- **Access**: Restrict access to `.env` file to authorized personnel only

### 2. Cryptographic Security

- **HMAC-SHA256**: Industry-standard cryptographic hash function
- **Constant-Time Comparison**: Prevents timing attacks
- **Payload Binding**: Signature includes payload hash
- **Timestamp**: Prevents replay attacks
- **Media Type Binding**: Prevents cross-media signature reuse

### 3. Backend-Only Verification

- **No Frontend Exposure**: Verification happens only on backend
- **No Secret Leakage**: Platform secret key never sent to frontend
- **Error Messages**: Generic error messages to frontend
- **Logging**: Detailed errors logged on backend for security monitoring

### 4. Attack Prevention

- **Tampering Detection**: HMAC verification detects any tampering
- **Replay Prevention**: Timestamp prevents signature replay
- **Cross-Media Prevention**: Media type validation prevents cross-media attacks
- **Version Control**: Version validation prevents downgrade attacks

### 5. Audit Trail

- **Comprehensive Logging**: All verification attempts logged
- **Failure Logging**: Failed verifications logged with details
- **Success Logging**: Successful verifications logged for audit
- **Processing Time**: Processing time logged for performance monitoring

### Common Issues

#### 1. Signature Not Found

**Error**: `Platform signature not found in media`

**Causes**:
- Media was not generated by SecureStego
- Media was generated before signature system was implemented
- Media was tampered with and signature removed

**Solution**: Ensure media was generated by SecureStego with signature system enabled.

#### 2. HMAC Verification Failed

**Error**: `HMAC verification failed - file may be tampered`

**Causes**:
- Media was tampered with after generation
- Platform secret key was changed
- Payload was modified

**Solution**: Verify media integrity and ensure correct platform secret key.

#### 3. Platform Mismatch

**Error**: `Platform identity mismatch - not generated by SecureStego`

**Causes**:
- Media was generated by a different platform
- Signature was copied from different media

**Solution**: Use media generated by SecureStego platform.

#### 4. Version Mismatch

**Error**: `Signature version not supported`

**Causes**:
- Media was generated with older version of SecureStego
- Media was generated with newer version not yet supported

**Solution**: Update SecureStego to support the signature version or use compatible media.

#### 5. Media Type Mismatch

**Error**: `Media type mismatch in signature`

**Causes**:
- Signature was copied from different media type
- Media type was changed after generation

**Solution**: Ensure media type matches the type used during generation.
